## Objetivo

## Tareas realizadas

- Se agregaron las dependencias JPA, MySQL y Flyway al archivo `backend/pom.xml`.
- Se configuró la anotación `@ActiveProfiles("test")` en los tests existentes (`InventoryApplicationTests` y `HealthControllerTest`) para asegurar la correcta carga del contexto utilizando la base de datos H2 en los tests.
- Se creó el script de migración base `V1__init_schema.sql` con el DDL completo para las tablas: `users`, `offices`, `materials`, `material_history`, `audit_log`, `refresh_tokens`, `api_clients`, y `api_client_scopes`, incluyendo llaves primarias, foráneas, tipos enum, marcas de tiempo auto-actualizables e índices obligatorios.
- Se validó la migración exitosa arrancando el contenedor Docker de MySQL y verificando que Flyway aplica correctamente el script.

## Problemas encontrados

- La versión para la dependencia `flyway-database-mysql` de Spring Boot 3.3.0 no estaba definida por defecto. Se reemplazó por la dependencia `flyway-mysql` que está bien gestionada por el POM padre de Spring Boot, solucionando el problema.
- Los tests integrados cargaban por defecto el perfil `dev` y fallaban al no encontrar el contenedor MySQL. Se solucionó configurando explícitamente `@ActiveProfiles("test")` en las clases de test.
