## Tareas realizadas

- **Docker Production Stack Setup**:
  - Creado el archivo `docker-compose.prod.yml` en la raíz del proyecto.
  - Configurado el servicio de base de datos `mysql_db` sin exposición de puertos al exterior para garantizar la seguridad.
  - Configurado el servicio `nginx` como proxy inverso y terminador de HTTPS en los puertos 80 y 443.
  - Definidos límites estrictos de recursos para todos los contenedores (`cpus` y `memory`).
  - Configurada la rotación de logs de Docker utilizando el driver `json-file` con límites de `10m` por archivo y un máximo de 3 archivos de retención.
- **Configuración de Proxy e HTTPS**:
  - Creado `docs/nginx/nginx.conf` implementando redirección automática de HTTP a HTTPS, configuraciones seguras de SSL (cifrados fuertes, protocolos TLSv1.2 y TLSv1.3 únicamente) y cabeceras de seguridad HTTP (`CSP`, `X-Frame-Options`, etc.).
  - Creado e integrado un script ejecutable helper `docs/nginx/generate-certs.sh` para la auto-generación de certificados SSL de desarrollo/pruebas.
- **Estrategia de Backups**:
  - Creado un script de backup en bash en [docs/backup.sh](file:///home/jmrs/gestionDeInventario2.0/docs/backup.sh) que ejecuta `mysqldump` dentro del contenedor MySQL, comprime el resultado y elimina respaldos con más de 7 días de antigüedad.
  - Documentado en la guía de despliegue la integración del script con el planificador `cron` del sistema.
- **Actualización de Documentación**:
  - Ampliado [docs/02-technical-structure/DEPLOYMENT_GUIDE.md](file:///home/jmrs/gestionDeInventario2.0/docs/02-technical-structure/DEPLOYMENT_GUIDE.md) con las instrucciones detalladas de despliegue en producción, obtención de certificados reales de Let's Encrypt con Certbot, rotación de logs y política de respaldos.
